테이블의 구조를 조회할 떄는 "DESC문을 사용한다." --> DESC문은 조회하는 테이블의 구조를 표형식으로 보여준다.

Field(필드): 테이블의 칼럼
Type(타입): 칼럼의 자료형 (int는 INTEGER의 별칭으로 돌다 정수형을 의미한다.)
Null(널): 칼럼에 값을 넣어야하는지 넣지 않아도 되는지를 나타낸다.(널은 값이 없음을 의미해서 yes라면 해당 칼럼에 값을 입력하지 않아도 문제가 되지 않지만 NO라면 반드시 값을 입력해야한다.)
Key(키): 칼럼이 테이블의 기본키(Primary Key, PRI), 외래키(Foreign Key, MUL), 고유키(Unique Key, UNI) 등 특별한 키로 사용되는지를
나타난다.
Default(디폴트): 칼럼의 기본값
Extra(엑스트라): 칼럼에 적용된 추가 속성

테이블 만들기: CREATE TABLE 테이블명 (
                칼럼명1 자료형1,
                칼럼명2 자료형2,
                ...
                PRIMARY KEY(칼럼명)
);
테이블 구조 조회: DESC 테이블명;
데이터 삽입: INSERT INTO 테이블명 (칼럼명1, 칼럼명2, ...)
            VALUES  (입력값1, 입력값2, ...);
데이터 조회: SELECT 칼럼명1, 칼럼명2, ...
            FROM    테이블명
            WHERE   조건;
데이터 수정: UPDATE 테이블명
            SET     칼럼명 = 입력값
            WHERE   조건;
데이터 삭제: DELETE FROM 테이블명
            WHERE   조건;
테이블/데이터베이스 삭제: DROP TABLE 테이블명;
                      DROP  DATABASE  데이터베이스명;
